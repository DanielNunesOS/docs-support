Architecture Dashboard 1.19

This is a preliminary version of the document.

New in Architecture Dashboard 1.19

  • Improved the permissions sync mechanism which now ensures the retry on the next sync if the permissions sync fails. (RADRT-2142)
  • The old Architecture Dashboard experience, which was still accessible to customers with old probes versions (older than 3.0) was discontinued. (RADRT-2148)

Bug Fixing

  • Fixed to have the correct object name on export the findings on the Report. (RADRT-2179)
  • Fixed tooltip that is showing when there is not enough data to show of solved findings for the selected range on Trend chard of Overview Dashboard. (RADRT-2193)
  • Fixed a bug when generating the data that impact on the number of findings on the Overview Dashboard which was wrongly counting with the solved facts on the open findings statistics. (RADRT-2195)
  • Fixed an issue that was wrongly showing all the teams for users that don't have permission to see it (RADRT-2197)
  • Fixed a bug that was wrongly solving all findings on a specific category when the customer had synchronization issues. (RADRT-2199)
  • Take into account the previous day data when showing the summary for technical debt on the Overview Dashboard (RADRT-2217)
