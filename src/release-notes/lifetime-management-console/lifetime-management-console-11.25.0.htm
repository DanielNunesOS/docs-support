LifeTime Management Console 11.25.0

Released on Oct 31, 2024

New in LifeTime Management Console 11.25.0

  • You can now check out the latest feature highlights that have recently been introduced to LifeTime on the new "What's New?" page. To access it, a new bell icon link has been added to the top right of the screen or it can be found at https:///lifetime/WhatsNew.aspx. (R11CT-3336)
  • The Platform Server version bundled with LifeTime has been upgraded to 11.30.0. (R11CT-3362)
  • LifeTime allows faster continuous delivery by having users add their team's applications to a deployment plan. This was inspired by Daniel Gonçalves' community idea. This accelerator is available by default and can be disabled/enabled by accessing the Settings screen of LifeTime. (R11CT-3471)
  • The Staging Edit screen now includes an application counter with the number of applications added to the plan. Inspired by João Heleno's idea. (R11CT-3526)

Bug Fixing

  • Fixed an issue that caused the list of affected consumers to display consumers that had already been deleted from the target environment when deploying applications with deleted producer modules. Reported by Matthias Preuter. (R11CT-3396)
  • Removed bug fix for RPM-4754 introduced in LifeTime 11.24.2 due to a known issue discovered when deploying modules moved between apps on LifeTime that lead to a deployment error. This applies for self-managed infrastructures. (R11CT-3551)
  • Fixed an issue that caused an error when trying to unregister an offline environment with maintenance mode enabled in an infrastructure with OIDC. (RPM-4854) Application Lifecycle LifeTime
  • Added a validation to the export sample data process which prevents an error from being triggered. This error occurred when applications with custom icons were exported as sample data during the SDK installation. Now, the added validation will skip incompatible images and the export process continues. (RPM-4935) Application Lifecycle LifeTime
  • Fixed an issue where the "Tag & Deploy" feature would generate tags with version numbers above the 3 digit limit, causing versions to be sorted incorrectly in the Application_History screen and LifeTime to fail to correctly identify the highest application version. This happened when the current application tag had a version number of 999, e.g. 0.999, and the application was marked for "Tag & Deploy" in the next staging. Now, when this happens, the next version number is incremented, e.g. 0.999+ is tagged as 1.0 instead of 0.1000. (RPM-5209) Application Lifecycle LifeTime
  • Fixed an issue where the /deployments/{DeploymentKey}/{Command}/ API would fail when trying to abort a plan. This happened when the target environment had the ServiceCenter site property DebugPerfShowUI set to true. (RPM-5223) Application Lifecycle LifeTime
  • Fixed an issue with the validation logic for the 'Deploy Now' button on staging. Depending on the settings of the parallel deployments feature, LifeTime incorrectly disabled the button in situations where it should have stayed enabled. (RPM-5283) Application Lifecycle LifeTime
  • Fixed ServiceStudio links in LifeTime to make them compatible with Chrome browser policies. (RPM-5292) Application Lifecycle LifeTime
