LifeTime Management Console 11.24.0

Released on Jul 03, 2024

New in LifeTime Management Console 11.24.0

  • Several improvements were introduced to the Reuse Plan use case. Now, the target environment from the completed plan is pre-selected as the source for the new plan; The new plan will use the same deployment notes as the original; The 'Keep the tags used in the original plan' is set as TRUE by default; and prompts were added to the source and target environment dropdowns. (R11CT-3017)
  • It is now also possible for self-managed infrastructures to execute in parallel multiple deployment plans that target a single environment. Previously this feature was only available for OutSystems Cloud. (R11CT-3161)
  • The Platform Server version bundled with LifeTime has been upgraded to 11.28.0. (R11CT-3162)

Bug Fixing

  • Fixed an issue that would show the 'Operation' tab and the 'deployment zone' dropdown while creating a staging even if only one Deployment zone is configured. Now, the 'Operation' tab and its content only show if there are more than one deployment zone. Inspired by Evert van der Zalm's feedback. (R11CT-3015)
  • Fixed an issue that showed overlapping application versions in the Deployment Plans list when the version was very long. Now the application versions are displayed correctly without overlapping. Inspired by Evert van der Zalm's feedback. (R11CT-3016)
  • Fixed an issue where the input to edit the description of an application version would be clipped. Now, the input is shown correctly without any clipping. Inspired by Nordin Ahdi's feedback. (R11CT-3020)
  • Fixed an issue that would allow the enablement of parallel deployments in infrastructures with only one environment. Now, the feature can only be enabled if the minimum required number of environments are available. Inspired by Nordin Ahdi's feedback. (R11CT-3050)
  • Fixed an issue which allowed the configuration of environment filters in infrastructures with only one environment. Now, environment filters are only available if the infrastructure has more than one environment. Inspired by Jorge Martins' feedback. (R11CT-3051)
  • Fixed an issue that allowed users to click "Create a new deployment plan" with only one environment available. Now, the button is disabled if there is only one environment available. Inspired by Nordin Ahdi's feedback. (R11CT-3052)
  • Fixed an issue that caused LifeTime to ask for the configuration of site properties incorrectly when using the option of 'Deploy Custom' for Traditional Web Applications. This occurred when modules that had new site properties were excluded from the deployment plan (using Deploy Custom) but LifeTime still asked to configure those site properties. Now, LifeTime only asks to configure site properties of the module versions that will be available in the target environment after the deployment is completed. Thank you Evert van der Zalm's and Jo√£o Martins for bringing this to our attention. (R11CT-3126)
  • Fixed an issue that would prevent the user from changing an application's deploy options after a producer application is automatically added to the plan. The issue occurred when a user with Access default role assigned to a team with Change and Deploy tried to deploy a consumer application and producer applications are automatically added to the plan. Now, the deploy options for all the applications in the plan can be changed. (RPM-4909) Application Lifecycle LifeTime
