LifeTime Management Console 11.22.0

Released on Apr 17, 2024

New in LifeTime Management Console 11.22.0

  • LifeTime API v2 now returns a new ‘DetailsURL' attribute in specific endpoints, namely GET Deployments, GET Applications and GET Environments. This new attribute is an URL to LifeTime that enables scenarios where there’s the need to access that same resource in the LifeTime user interface once fetched from the API - for instance, to set deployment plan configurations related to integration endpoints or database catalogs. (R11CT-2907)
  • The Platform Server version bundled with LifeTime has been upgraded to 11.26.0. (R11CT-2908)
  • From now on, one can manage LifeTime specific settings directly in the UI, for improved autonomy. Managing the data retention of different cleanup mechanisms is an example of this. The screen can be found at [https://hostname/lifetime/Settings.aspx|https://hostname/lifetime/Settings.aspx]. (R11CT-2910)
  • It is now possible to do deployments across infrastructures by using LifeTime API v2. This API has 4 new endpoints that allow IT users to perform binary (e.g. based on OAP and OSP files) deployments between different infrastructures. The newly added endpoints are:
    • POST /environments/{EnvironmentKey}/solution
    • GET /environments/{EnvironmentKey}/solutionstatus/{SolutionKey}
    • GET /environments/{EnvironmentKey}/solution/{SolutionKey}
    • POST /environments/{environmentKey}/deployment
    For more details on how to use them, please check the documentation here. (R11CT-2934)
  • LifeTime now cleans up old deployment operations to keep a good performance. By default, LifeTime will keep 90 days of data. The data retention period can be configured in [[LifeTime Settings page|https://hostname/lifetime/Settings.aspx|https://hostname/lifetime/Settings.aspx|https://hostname/lifetime/Settings.aspx]]. The clean up process will start a week after upgrading. The delayed start allows infrastructure managers to configure a different data retention period if they believe that is necessary. (R11CT-2935)

Bug Fixing

  • You can now use the LifeTime API GET /deployments/{DeploymentKey}/status/ to get the type of deployment messages. Possible returned values are Info, Warning, and Error. (R11CT-2903)
  • Removing an application from a deployment using LifeTime API effectively removes it from the deployment’s application list. Previously, when using LifeTime API to update a deployment plan and remove an application, that application would be set as ‘Do Nothing’ and wouldn’t be effectively removed from the plan. Now, the removed application will not be part of the deployment’s application list. This is a breaking change in the method’s behaviour. (R11CT-2939)
  • Improved the performance of deployment preparation by reducing significantly the number of requests made to retrieve site property information (used in the settings screen) from the target environment. The performance degradation was initially introduced with LifeTime version 11.20.1. (RPM-4813) Application Lifecycle LifeTime
  • Resolved an issue where cloud customers were unable to modify configuration settings to enable or disable the parallel deployment feature. (RPM-4882) Application Lifecycle LifeTime
