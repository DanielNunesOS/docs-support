LifeTime Management Console 11.17.0

Released on Apr 21, 2023

New in LifeTime Management Console 11.17.0

  • Due to poor adoption, the 9 dot menu with navigation links for documentation and other product parts, present in the upper right corner of all screens, has been removed from LifeTime. (R11CT-2031)
  • The Built-in authentication for IT users now has a password complexity rule to ensure stronger security against potential hackers. The rule enforces a 12-character minimum length and at least one number, uppercase and lowercase letter. Having no effect on old passwords, the rule will apply when creating or updating passwords for IT users, enhancing the platform's overall security. (RDOI-236)

Bug Fixing

  • Fixed an issue that caused deleted modules to be tagged as incompatible during staging. This occurred when the producer module was changed and would be incompatible with the consumer module in the target environment. Now, consumer modules that will be deleted during the staging no longer cause incompatibility errors with the producer module. (RPM-1186) Application Lifecycle
  • Fixed an issue with the device library, which would identify MS Edge as Chrome in LifeTimeAnalytics. This occurred for every user request using MS Edge. Now, Edge usage is displayed correctly. (RPM-2157) Application Lifecycle LifeTime
  • Improved validation of service account user permissions during a sync. During the synchronization of the applications with the environments, this user was having a problem with permissions being denied, affecting the synchronization of the applications in some scenarios. (RPM-2390) Application Lifecycle LifeTime
  • Fixed an issue that caused performance problems in infrastructures with limited network connectivity when fetching the links present in the upper right corner of all screens in LifeTime. (RPM-2797) Application Lifecycle LifeTime
  • Fixed a bug that caused special characters to appear in a feedback message. This occurred when changing an IT User default role. (RPM-3162) Application Lifecycle LifeTime
  • Fixed a bug that did not apply entity settings defined during Staging Settings when the extension with an external database was deployed to the target environment the first time. (RPM-3542) Application Lifecycle
  • Fixed an issue that triggered sync processes for unregistered environments and caused log pollution with "Invalid username or password" error. (RPM-3727) Application Lifecycle LifeTime
  • Now, when modules are identified to delete in a Staging, the warning message includes information about the consumer modules that may have runtime errors in the target environment, and user action is required to proceed and unlock the "Deploy Now" button. (RPM-3746) Application Lifecycle LifeTime
  • Fixed an issue on synchronization that could lead to applications not being visible for the LifeTime users. (RPM-3775) Application Lifecycle LifeTime
  • Fixed an issue on long running staging plans that would lead to client error due to the size of viewstate. Now a full page load is issued from time to time following the same approach of Applications list page when open. (RPM-639) Application Lifecycle LifeTime
  • Fixed an issue that caused the Authentication Settings screen to crash when opened. This occurred when the IIS of one of the environments connected to LifeTime was down. Now, the screen will open and show a feedback message warning that a problem occurred while opening the screen. (RPM-692) Application Lifecycle LifeTime
