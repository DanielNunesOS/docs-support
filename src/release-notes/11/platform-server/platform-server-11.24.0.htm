Platform Server 11.24.0

Released on Nov 13, 2023

New in Platform Server 11.24.0

  • Deploy Service executable and related files are now isolated inside {Platform Server folder}/DeployService/ folder, instead of the root install path. This will improve maintainability of that service. (R11BRT-690)
  • Added the ARIA alert role to the feedback message in order to improve accessibility with screen readers. Inspired by Joost Miltenburgâ€™s idea. (R11DT-2046)
  • It is now possible to integrate with SAP services using the SAP Connector for Microsoft .NET 3.1. Using previous versions of the SAP Connector will not be supported. Check OutSystems 11 side effects and breaking changes for more details. (RO11IT-592)

Bug Fixing

  • Released a toggle to disable the sanitization of valid numbers that begin with -/+ signs when exporting RecordListToExcel. The sanitization will stay enabled by default (e.g. -1 is sanitized to '-1). To disable the sanitization, one needs to set the ossys_parameter OutSystems.HubEdition.SanitizeSpreadsheetNumbers to False, restart Deployment Controller Service, and Apply Settings to the Factory. (RPM-1206) Application Runtime Logic Execution
  • Reactive File Upload widget is now focusable. A user can navigate through screen components by pressing Tab until File Upload is focused. Then they can press Enter or Space keys to open File Explorer and choose the file to upload. For applications that use OutSystemsUI as their base theme, OutSystemsUI 2.18.0 will introduce new CSS to highlight the widget when it's focused. It will also extend the Accessibility Features to this widget. (RPM-1899) Application Runtime Interface
  • Feedback Message is now focusable. A user can navigate through screen components by pressing Tab until Feedback Message is focused. Then they can press Enter or Space keys to close the message. (RPM-2007) Application Runtime
  • Fixed an issue that caused screen readers to incorrectly access screen content when multiple popups exist on the screen. This applies only to reactive web apps. (RPM-2480) Application Runtime
  • Fixed an issue that would lock the database when logging in with the same credentials, using the OutSystems basic authentication API. This would occur when the Login method is called repeatedly with the same credentials for AD or LDAP users, causing a database record lock. This lock would hold the login process and register an error even though the first request was successful. Now, only the required database queries are performed, avoiding the lock altogether. (RPM-2524) Application Lifecycle Users
  • Fixed an issue that caused the attribute aria-activedescendant of a Button Group widget to be incorrectly populated. This would raise an accessibility issue reported by an accessibility verification tool. (RPM-2702) Application Runtime Interface
  • Fixed an issue that caused a memory leak when navigating between or refreshing reactive or mobile screens.
    In screens that had many UI components, this could cause the app to restart once it reached a specific memory threshold.
    This occurred because some UI components were not unregistering from the LocaleService, upon being removed from the UI, causing them to be kept in memory.
    (RPM-3540) Application Runtime
  • Now, when exporting the "Timers Performance" report from the Service Center analytics, the report will have the module and timer names instead of their identifiers. (RPM-3695) Application Lifecycle Service Center
  • Added more control over the Deadlock Priority of `BPT_API`'s `Process_BulkDelete` . Deadlock Priority can be lowered by toggling the new `OSSYS_PARAMETER` `OutSystems.RuntimePublic.ProcessDeletion.DeadlockMitigation`. (RPM-3954) Application Runtime Processes
  • Fixed an issue in Service Center that allowed users with List permission to access or edit information in the Application Detail screen. This issue occurred when an application had its native distribution set up and a user with List permission navigated to the Distribute tab on that application's detail page. Now, every link or input is disabled for that page if the user doesn't have the right permissions. (RPM-4121) Application Lifecycle Service Center
  • Fixed an issue that prevented users from assigning Database Connections to Logical Databases when configuring extensions.
    This occurred when there were more than 100 Database Connections configured and the user didn't have access to the majority of them.
    Now, assigning Database Connections to Logical Database is independent of the number of existing connections. (RPM-4134) Application Lifecycle Service Center
  • Fixed the Service Center's search functionality by ensuring it correctly encodes special Unicode characters in its database SQL queries. This enhancement addresses the issue where applications with names written in Japanese characters, for instance, couldn't be properly searched for on the Service Center's applications list screen. (RPM-4186) Application Lifecycle Service Center
  • Fixed an issue when loading Excel files that use named ranges using the ExcelToRecordList functionality. New behaviors of certain Excel patterns are now considered. This fix implies a breaking change. Check OutSystems 11 side effects and breaking changes for more details. (RPM-4332) Application Runtime Logic Execution
  • Fixed an issue on Reactive Web and Mobile apps that caused the error "Error Opening Popover: Cannot read properties of null" to be shown when opening a PopOver widget inside a PopUp widget. (RPM-4336) Application Runtime Interface
  • Fixed an issue in which an error message related to native platforms was inconsistent with the corresponding tab in the module detail page of Service Center. Now, when a module is changed within a mobile application, Service Center will display the following message: "The mobile app must be tagged in LifeTime and generated if you want to submit it to the mobile app store. Check Distribute tab for more details."; which is consistent with the title of the tab in the espace detail screen. (RPM-4346) Application Lifecycle Service Center
  • The issue related to SAML login due to the exception "Cannot insert duplicate key row in object 'dbo.OSUSR6WDUSERSESSION' with unique index 'OSIDXOSUSR6WDUSERSESSION16SAMLSESSIONINDEX9TENANTID'" has been fixed. End-users were unable to login if they have multiple active sessions already. While doing a fresh login a new session object was being created with an existing session id that resulted in duplicate key row exception (RPM-4371) Application Lifecycle Service Center
  • Fixed an issue in Integration Studio that caused database tables to not be shown when connecting to external databases in an extension. This occurred for users with LifeTime permissions lower than "Open and Debug" for the environment. In addition, only tables that were already being used in other modules or extensions in the environment were affected. (RPM-4393) Application Lifecycle Service Center
  • Fixed issue on RT cookie settings when LifeTime Analytics is on for the environment. (RPM-839) Application Runtime Cookies
  • Fixed an issue that would cause 2-stage deployment plans to finish with errors. This issue occurred when 'Continue Deployment' was clicked after the solution associated with the plan was deleted in the target environment during the execution of the SolutionPack_Cleanup timer. Now, LifeTime will receive the information that the solution was deleted and will abort the deployment plan. (RPM-969) Application Lifecycle LifeTime
  • Fixed an issue that allowed a user without publish permission, to deploy a module in Service Studio when the same module version is already published in the server and, therefore, the upload is skipped. CVSSv3.1 score 3.3 (Low). (RPM-3215) Publish Operation Upload
  • Fixed an issue that allowed cloud customers to change the name of an environment in the screen Environment_Configuration of Service Center. CVSSv3.1 score 2.7 (Low). (RPM-4329) Application Lifecycle Service Center

Breaking change

  • Fixed an issue when loading Excel files using the ExcelToRecordList functionality. New behaviors of certain Excel patterns are now considered. Check OutSystems 11 side effects and breaking changes for more details.
