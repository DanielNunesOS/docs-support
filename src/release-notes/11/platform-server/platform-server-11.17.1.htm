Platform Server 11.17.1

This is a preliminary version of the document.

New in Platform Server 11.17.1

  • Now with IDP activated, the User will be redirected to the IDP page immediately on entering accessing Lifetime or Service center skipping the login screen. (R11CT-1386)
  • Now, when the infrastructure has the IdP service active, users from the default user pool will no longer be able to log in using a username and password. (R11CT-1387)
  • 1. Now with IDP activated, the User will be redirected to the login screen after clicking on Logout with username and password fields hidden and the only available option will be "Log in with SSO" 2. There was a change in the label of the button from "Login" to "Log in" (R11CT-1422)
  • Raised the required Microsoft .NET Windows Server Hosting from 3.1 to 6.0. The checklist and documentation instruction links have been updated to reflect this. (R11PIT-809)

Bug Fixing

  • Improved email processing time when there are delays sending or fetching emails from the database. (RPM-1341) Application Runtime Interface
  • Increased the length of the attributes (SessionIndex, MessageID and ResponseToMessageID) in Users SamlMessage_Log entity (RPM-2463) Application Lifecycle Users
  • Fixed cross-site scripting vulnerability in Service Center. CVSSv3.1 score 5.4 (Medium). (RPM-2827) Application Lifecycle Service Center
