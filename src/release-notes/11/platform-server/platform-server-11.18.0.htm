Bug Fixing

  • Fixed a lock while updating Site Properties on long requests. (RPM-1032) Publish Operation
  • The parameter "OutSystems.HubEdition.OracleSequenceCacheSize" was created in the OSSYS_PARAMETER table to control Oracle sequence cache size. The default value is 100 and the acceptable range of values is between 2 and 1000 (inclusive). If the parameter does not exist, the cache size for new Oracle sequences will use the default value. If the parameter exists and its value is less than 2, Oracle sequences will be created with "NO CACHE". If the parameter exists and its value is greater than 1000, the cache size for new Oracle sequences will be 1000. If the parameter exists and its value is between 2 and 1000, the cache size for new Oracle sequences will be the parameter value. When upgrading the database to version 11.18.0 by running the Configuration Tool, existing sequences that have no cache will be updated to have the value defined in the parameter (or the default).
