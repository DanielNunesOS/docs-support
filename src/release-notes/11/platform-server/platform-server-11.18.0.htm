Platform Server 11.18.0

This is a preliminary version of the document.

New in Platform Server 11.18.0

  • It is now possible to see the site property description in the table on the Site Properties list screen. Inspired by Takasi Moriya and Raphael Laurence Reyno's ideas. (R11CT-308)
  • It is now possible to filter the screens logs by User. Inspired by Luís Figueiredo's idea. (R11CT-311)
  • Added support for the redaction of header and body parameters of an exposed REST API when the Logging Level is set to Full or Troubleshoot. (R11DT-1118)
  • Added the ability to deploy all modules prepared during an upgrade. It is now possible to optionally, after upgrading your OutSystems environment to Platform Server 11.18.0+ version, to immediately deploy all modules that were successfully prepared. If executed, the "Deploy All" action will reflect all changes and improvements from the new Platform Server version in the applications running code.
    We recommend that customers skip this step and instead publish modules and/or apps manually as needed. Nevertheless, for those customers/scenarios where a deploy of all upgraded applications is needed ASAP, customers now have the option to do it.
    For more details about the deployment of prepared modules check the documentation. (R11PIT-885)
  • Solution publishing was optimized and is now up to 40% faster in large factories. (R11PT-80)
  • Applies only to environments using an Oracle database.
    Now, new sequences used for auto numbers created by the Platform have a cache size of 100.
    When upgrading the Platform to version 11.18.0 by running the Configuration Tool, existing sequences that have no cache will be updated to have a cache size of 100. (RPM-1158) Infrastructure Management Cloud Orchestration
  • OutSystems Deployment Controller and OutSystems Scheduler Services now run with Virtual Accounts. If you have permissions errors when installing please check the following documentation. (RPM-1334) Infrastructure Management OutSystems Services
  • It is now possible to read and display data from PostgreSQL external databases columns with the 'Time (without TimeZone)' data type . (RPM-3043) Application Runtime Data Access and Manipulation

Bug Fixing

  • Fixed an issue that prevented some licensing errors from displaying the correct error message. (R11BRT-152)
  • Fixed an issue that caused a failed SAP connection when clicking the "Test Connection" button in Service Center. (R11DT-1452)
  • Improved performance of Pool Garbage Collector job (R11PIT-778)
  • Updated installation checklist to add information regarding the need to have "Create Symbolic Links" permission when running the configuration tool. (R11PIT-784)
  • Added disclaimers relating to the use of the "Create/Upgrade Service" in custom RabbitMQ installation scenarios in Configuration Tool and Installation Checklist. (R11PIT-807)
  • Fixed an issue that caused Service Studio to assume the wrong module version as the most recent when using Oracle RAC databases. (R11PT-24)
  • Fixed a lock while updating Site Properties on long requests. (RPM-1032) Publish Operation
  • Fixed an issue in Factory Configuration that prevented Shared Configurations from being saved and caused an error related with Administrator permissions.
    This occurred when shared configurations were associated with deleted Modules. (RPM-1175) Application Runtime Supported Forge Component
  • Added logs to provide additional information when deployment service fails. (RPM-1202) Infrastructure Management OutSystems Services
  • Fixed an issue that occurred after creating a foreign key that caused database locks with wait type "enq: TM – contention" and a significant increase in the database CPU usage.
    This happened in environments using an Oracle database. (RPM-1609) Publish Operation
  • Fixed an issue in LifeTime that caused deployment plans to fail due to lack of user permissions when no new reference was added. (RPM-1689) Application Lifecycle LifeTime
  • Fixed an issue that caused modules and apps to have different settings in different servers of the same Deployment Zone.
    This occurred after adding more servers to a Deployment Zone and then changing settings of a module or app that was deployed before the new server was added. (RPM-1830) Application Runtime
  • Fixed an issue that caused a deployment to fail with an ORA-01440 error in environments using an Oracle database.
    This occurred when deploying a module after decreasing the precision or scale of an entity's attribute, by changing its Data Type, or the Length or Decimals properties.
    Now, compilation checks whether a change to an attribute's data type would raise the ORA-01440 error. (RPM-2042) Publish Operation
  • Fixed an issue that caused Service Studio to assume the wrong module version as the most recent when using Oracle RAC databases. (RPM-2220) Publish Operation Deployment stage
  • Fixed an issue that caused different module runtime behavior depending on the Debug Mode for the module. Learn more about environment and module Debug Mode.
    This occurred in actions that included asynchronous logic and that were set as event handlers. For example, this affected the behavior of the Button Loading widget. (RPM-2269) Application Runtime Logic Execution
  • Improved NLS_Language tooltip's text for Extenal Databases configurations. (RPM-2398) Application Runtime Data Access and Manipulation
  • Fixed a bug with Download Node which caused alternative logic branches from a Client Action to be executed. (RPM-2410) Application Runtime Logic Execution
  • Fixed an issue that caused runtime errors when authenticating users using the SAML authentication available in the Users app.
    When trying to authenticate users with SAML at runtime, there were "Unable to Process Request" errors. Checking the logs in Service Center, there was an error message similar to "String or binary data would be truncated in table 'OSDev.dbo.OSUSR_6WD_SAMLMESSAGE_LOG', column 'SESSIONINDEX'. Truncated value: '_x18MaTZNE2aO1vm5_xMcXivDMh0LRTrbxS0uuDPV3SOB1261'. The statement has been terminated." Alternatively, the message might mention the MessageID column instead.
    To solve this issue, the length of the SessionIndex, MessageID, and ResponseToMessageID attributes in the Users SamlMessage_Log entity was increased. (RPM-2463) Application Lifecycle Users
  • Updated installation checklist to add information regarding the need to have "Create Symbolic Links" permission when running the configuration tool. (RPM-2549) Infrastructure Management Platform Installation Checklist
  • Fixed an issue where front-end servers with ip "::1" would display wrong status information under the "Environment Health" when stopping deployment service (RPM-2711) Infrastructure Management OutSystems Services
  • Fixed an issue that caused extensions using the PostgreSQL External Database Connector to fail saving data when a column of data type 'uuid' was included. (RPM-2844) Application Runtime Data Access and Manipulation
  • Fixed exception when compiling a module with a reference to a Service Action with a deleted parameter with a default value (RPM-2848) Application Lifecycle LifeTime
  • Fixed an issue that caused a wrong translation in multiple languages. This occurred in Japanese, Chinese and Dutch for the translation of "Username". (RPM-2857) Application Runtime Interface
  • Fixed an issue where Server API incorrectly sends the catalog Name instead of the catalog Logical Name to Service Center when publishing solutions or applications. (RPM-2860) Infrastructure Management
  • Fixed an issue that caused a failed SAP connection when clicking the "Test Connection" button in Service Center. (RPM-2871) Application Lifecycle Service Center
  • Improved telemetry collection for Progressive Upgrade errors (RPM-2889) Infrastructure Management
  • Fixed an issue that prevented customers from proceeding with an ongoing upgrade due to an unusual longer initialization of Compiler service. (RPM-3126) Infrastructure Management Platform Configuration
  • Fix a bug that was leading to compilation errors in applications with SOAP Web Services. (RPM-3176) Publish Operation Compilation
  • Fixed an issue in SynchronizeDomainUsers action that was causing it to fail if a given user that was set as active in the Platform did not exist in Active Directory. (RPM-477) Application Runtime Authentication and Authorization
  • Fixed an issue that prevented Deploy Service and Scheduler from starting when the database is down. (RPM-787) Platform Installation OutSystems Services
  • Fixed an issue that caused Auto Number values to not be incremented on non-Identifier attributes when using Oracle databases. (RPM-882) Application Runtime Data Access and Manipulation
  • Fixed an issue where outdated scripts would be executed when republishing a module that had failed during deployment. Now compilation files will be deleted after a deployment error to ensure they aren't reused in the following deployment. (RPM-957) Platform Installation Configuration
  • Fixed a security vulnerability. CVSSv3.1 score 10 (Critical). (RPM-3055) Infrastructure Management
  • Fixed a security vulnerability. CVSSv3.1 score 9.8 (Critical). (RPM-3056) Infrastructure Management
