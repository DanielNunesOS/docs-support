Platform Server 11.29.0

Released on Jul 22, 2024

New in Platform Server 11.29.0

  • Upgraded Npgsql (PostgreSQL library) to version 5.0.18. It includes a security vulnerability fix introduced by those versions. It only affects Platform Server installations using PostgreSQL external database connections. (R11BRT-1010)
  • Added the supported compatibility levels for Azure SQL in the installation checklist. (R11BRT-1151)
  • Fixed the description of Database_WriteInParallelTransaction of the PlatformRuntime_API extension to explain better what is a parallel transaction. (R11BRT-344)
  • Added support for Amazon EC2Launch v2. System requirements documentation page was changed to reflect this change. (RPM-4973) Infrastructure Management

Bug Fixing

  • PostgreSQL external database connections correctly handle the "time with time zone" data type. (R11BRT-1050)
  • Fixed the way the Stack field of some Error Logs was being generated to ensure the message of the exceptions was not lost. In some situations the message field of the Error Log was also being repeated in the beginning of the Stack field. That repetition was removed to make it consistent with the remaining logs. (R11BRT-1061)
  • Removed <Ignore Field> value as a possible NameId format in SAML configuration. (R11BRT-1063)
  • Configuration Tool no longer allows conflicting ports for OutSystems services and only numeric values between 0 and 65535 are allowed. Configuration Tool in GUI mode and unattended mode now warns the user when invalid values are set in the server configuration file (server.hsconf). In GUI mode default values are loaded for the entire configuration. In unattended mode the execution is aborted. (R11BRT-1071)
  • Updated installation checklist and public documentation to better explain the requirements for architectures with dedicated Deployment Controller machines in regard to deployment zones. Part of this problem was also already fixed in the previous version 11.26.0 in issue R11BRT-821 regarding other architectures configurations. (RPM-2415) Infrastructure Management Platform Configuration
  • Fixed issue in SAML authentication where NameID format was not being correctly sent in Logout Request leading to issues with Active Directory Federation Services integration as an IdP. (RPM-3827) Application Runtime Authentication and Authorization
  • Fixed an issue that occurred when two users persistently logged in consecutively without an explicit logout between them. Previously, this caused the second user to lack a persistent session, while the first user's session remained active. Now the first user's session is properly terminated and the second user has a new persistent session. This applies to Traditional apps only. (RPM-4428) Application Runtime Authentication and Authorization
  • Fixed an issue that caused client-side error messages to sometimes contain internal database details. This occurred when running server-side database operations in reactive, mobile and traditional web apps. Now, database errors show a generic client-side error message that can be customized using the setting 'Database Default Error Message' from Factory Configuration, while the original error is logged in Service Center. (RPM-4429) Application Runtime
  • Fixed an issue with static entities with locales and 'Use Translations' set to 'No' throwing runtime errors when consumed in a module of type Library (RPM-4748) Application Runtime Data Access and Manipulation
  • Improved resilience of analytics reports generation when Log Separation is configured. After the upgrade there will be retries for older reports that already potentially expired and will fail to be created. It will log errors for a while until it finishes all the retries for the old reports. (RPM-4812) Application Lifecycle Service Center
  • Fixed an issue that caused repeated "Executed part of the script [modulename]/[scriptname].sql" General log messages when publishing a module. Now, these messages won't be recorded anymore. (RPM-4837) Publish Operation Deployment stage
  • Fixed the internal log classification for Log Streaming that was causing relevant app logs not to be streamed. The expected behavior is that logs from the system, such as those from the Service Center, would not be streamed while still keeping app logs. To mitigate this issue, all logs, including system logs, are currently being sent. It's expected that in the future the expected behavior will be implemented (RPM-4916) Extended Product Logs Streaming
  • Fixes incorrect warnings on LifeTime and Service Center pages where certain extensions (namely Integration Builder's) would show as needing to be upgraded to the latest platform server version. (RPM-4974) Service Studio
  • Managing Internal Network settings, for OutSystems Cloud infrastructures, no longer requires restarting manually platform services. (RPM-4985) Application Lifecycle Service Center
  • Improved the performance of the Deployment Service by optimizing the validation of unused folders. (RPM-4995) Infrastructure Management OutSystems Services
  • Fixed the generation of the Daily Activity analytics reports in Service Center when there is a large number of sessions on infrastructures using the Log Separation feature. (RPM-4997) Application Lifecycle Service Center
  • Fixed the installation checklist to have the Service Studio and Integration Studio as mandatory steps on both First Install and Upgrade configurations. Previously when Upgrade was selected, the steps only mentioned them for Deployment Controller machines. While they are not normally used from other servers, it is important to keep their installation consistent in case of operations like a Deployment Controller failover needs to be performed. (RPM-5015) Infrastructure Management Platform Installation Checklist
