Cross-platform Service Studio 11.54.1

This is a preliminary version of the document.

New in Cross-platform Service Studio 11.54.1

  • Improved attribute mapping suggestions to consider the target attribute name. Inspired by feedback from Carlos Olías. (RAID-1687)
  • Improved the Merge experience so that your input will only be needed when there are actually conflicts that need resolving. When possible, the merge will happen automatically during the publish process and not need your intervention as before. Merges with yourself will no longer occur in situations where you are publishing code which started off in an earlier revision. As long as no other user has published code, and you are the last person to have published, your changes will always overwrite the code, avoiding merges with yourself. (RDOIAT-970)
  • It is now possible to move module tabs between existing Service Studio windows. (RDOIBT-1085)
  • You can now have up to 12 module tabs simultaneously opened in a Service Studio window. (RDOIBT-1102)

Bug Fixing

  • Fixed an issue that caused links to be incorrectly rendered in Comment nodes. (RDOIBT-1214)
