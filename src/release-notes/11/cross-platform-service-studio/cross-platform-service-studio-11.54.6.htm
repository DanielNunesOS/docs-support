Cross-platform Service Studio 11.54.6

This is a preliminary version of the document.

New in Cross-platform Service Studio 11.54.6

  • Now, applying Undo(s) to rollback changes on top of the published version(s) will no longer generate conflicts or automatic merges with the version(s) published from that module Tab, and instead consider the new version as an evolution of the published one. (RDOIAT-1042)

Bug Fixing

  • Fixed an issue that caused Identifier attributes in Inline Records to show suggestions that didn’t correspond to the desired type and not show expected suggestions, such as ‘NullIdentifier()’. This occurred because Identifier typed attributes were using their basic type rather than their actual type. Reported by [Kilian Hekhuis|https://www.outsystems.com/profile/zbx7fxyfxj/overview]. (RAID-1871)
  • Fixed issue that caused attribute mappings to not be automatically filled in some scenarios where the attribute names and data types completely match. (RAID-1900)
  • Fixed an issue that happened when comparing with another version or file, which would not respond to navigation double clicks in the merge editor differences trees.  (RDOIAT-1058)
  • Fixed an issue that caused a failure connecting to Chrome when starting Debugger when another Chrome instance was left open from a previous debug session. (RDOIAT-1060)
  • Fixed an issue that caused a misleading warning to show up during the installation of Forge components, also requiring the user to do a forced installation, in scenarios in which the module dependencies had only optional differences from the one in the user environment. (RDOIBT-1259)
